### Incident: travamento no boot (resolvido)

**Sintoma:** após o primeiro push (commit `cbdb9c7`), a página ficava travada em "Verificando sessão…" sem nunca abrir o login nem o shell.

**Causa raiz:** o app.js original tinha 3 `return;` cedo no boot pra abortar se `TERRA_CONFIG` ou o SDK do Supabase faltassem. Funcionavam porque estavam **dentro da IIFE**. Sem a IIFE wrapper, `return` no top-level de um `<script>` de browser é **SyntaxError silencioso** — script 01 não parsa, cliente Supabase nunca é criado, boot trava.

**`node --check` não detectou** porque Node trata scripts como CommonJS module (com função-wrapper invisível). Pra validar isso direito tem que usar `vm.Script(content)` que parsa como SCRIPT GLOBAL, exatamente como o browser.

**Fix (commit `c4e11e8`):** os 3 returns viraram flag `_terraBootOK` + if/else aninhado. Bloco do auth flow também envelopado em `if(_terraBootOK)` pra não tentar `client.auth.X` com client null.

**Lição:** ao remover IIFE wrapper de qualquer script JS, procurar TODOS os `return` top-level (não dentro de função) e convertê-los pra estrutura sem return. Validar com `vm.Script` ou navegando direto no browser, NÃO confiar só em `node --check`.
