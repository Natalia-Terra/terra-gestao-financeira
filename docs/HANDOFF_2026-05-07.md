## 4. 🚨 BUG CRÍTICO #1 — Reset Completo bloqueia interações

### Sintoma
Na tela **Configuração > Reset Completo**, o input "Digite RESET" não aceita texto e o botão "Executar Reset Completo" não é clicável. A Juliana e a Natália testaram em **navegadores diferentes** (Chrome e Edge) — mesmo problema. Não é cache local.

### O que JÁ foi tentado e NÃO resolveu
- `db061ec`: adicionei regra CSS `.modal-overlay[hidden] { display: none !important; }`. Confirmei que está em produção via curl. Mas não resolveu.
- A regra `[hidden] { display: none !important; }` (linha 70 do styles.css) já existia antes — também é !important.

### Causa raiz mais provável (ainda não confirmada — investigar na próxima sessão)

Existem **3 lugares no código que criam elementos `.modal-overlay` sem o atributo `hidden`** e adicionam ao body:

1. **`mostrarMensagem()`** em app.js linha ~822 (helper que adicionei hoje, commit `b80a813`) — cria `<div class="modal-overlay">` SEM hidden, usa `div.remove()` ao fechar.
2. **`abrirModalDetalhe()`** em app.js linha 7805 (já existia antes) — cria `<div class="modal-overlay" id="modal-detalhe-overlay">` SEM hidden, remove via `parentNode.removeChild()`.
3. (Possíveis outros não localizados ainda.)

Quando esses modais são fechados, **deveriam** ser removidos do DOM. Mas se algum erro JS impede a remoção (ex: exceção dentro do handler de Fechar), o `<div class="modal-overlay">` fica "órfão" no body, sem `hidden`, e portanto **a regra `[hidden] !important` não se aplica**. Resultado: overlay invisível com `z-index: 50` cobrindo a app inteira, absorvendo cliques e impedindo interações.

### Como investigar na próxima sessão

1. **Reproduzir o bug** abrindo o site, indo direto pra Reset Completo SEM clicar em mais nada antes — ver se já travado.
2. Se SIM (bug existe sem nenhuma interação prévia), o problema é **provavelmente um modal-detalhe ou similar criado durante o boot/render que não foi removido**.
3. No DevTools (F12), aba **Elements**, fazer Ctrl+F e buscar por `class="modal-overlay"`. Listar todos os elementos com essa classe e seu estado (hidden? display?).
4. Verificar se algum `addEventListener` em `cfg_reset` faz algo que cria modal-detalhe.

### Solução defensiva (recomendada implementar logo)

No boot do app (logo após autenticação) e/ou na função `carregarResetSeNecessario`, adicionar:

```javascript
// Self-healing: remove modal-overlays órfãos sem hidden
document.querySelectorAll('.modal-overlay:not([hidden])').forEach(function (m) {
  if (!m.dataset.terraVivo) m.remove();
});
```

Onde `data-terra-vivo` é uma flag que TODA função `abrirModal*` deve setar quando cria seu modal e remove ao fechar. Assim qualquer overlay órfão é faxinado automaticamente.
